## How permissions work

By default, destructive operations require confirmation before execution. You can configure rules to customize this behavior per tool.

## Managing permissions

```bash
/permissions
```

## Permission types

<Properties>
  <Property name="ask" type="default">
    Always ask before executing. This is the default for destructive operations like file writes, deletes, and shell commands.
  </Property>
  <Property name="allow" type="rule">
    Always allow without asking. Use for tools you trust completely.
  </Property>
  <Property name="deny" type="rule">
    Never allow execution. Block specific tools entirely.
  </Property>
</Properties>

## Headless mode

Use `--force` to auto-approve all operations:

```bash
ai -p --force "delete all temp files"
```

<Callout type="warning">
  The `--force` flag bypasses all permission checks. Every tool call is auto-approved, including file deletes and arbitrary shell commands. Only use this in trusted environments.
</Callout>
